1.1 · What context costs14
Life of an Agent · S1—
Why long agent runs get expensive fast

Attention cost grows quadratically in context length.

the cost shape
Doubling the context window doesn't double the matrix multiplies per token — it roughly quadruples them. Self-attention is O(n²) in sequence length.
In practice: prefix caching, sliding windows, and chunked attention soften this. The shape doesn't change.
what this implies
A 200K-token turn isn't 6× more expensive than 32K. It's closer to ~40×.
An agent that lets its context grow unbounded is buying a quadratic bill.
Compaction and trimming aren't UX polish — they're cost control.

1.2 · Reading a model card17
Life of an Agent · S1—
The vocabulary you'll see on every spec sheet

Four numbers, one date.

context window
How many tokens fit in one call. 128K, 200K, 1M. The model's working memory.
total parameters
Weights on disk. What it costs to host.
active parameters
Weights that fire per token. What you pay in matrix multiplies. Equal to total only on dense models.
sparsity
total ÷ active. A 30× ratio means 1 in 30 weights does work per token.
knowledge cutoff
Last date in pre-training data. Everything after is invisible to the base model — unless you put it in the context window.
// We'll unpack each one. Active vs total is where most people's mental model is wrong.
